容器管理

容器简介

容器是什么？

容器就类似于我们运行起来的一个操作系统，而且这个操作系统启动了某些服务。

这里的容器指的是运行起来的一个Docker镜像。

查看、启动容器

查看容器

# 命令格式
docker ps

注意：

管理docker容器可以通过名称，也可以通过ID

ps是显示正在运行的容器， -a是显示所有运行过的容器，包括已经不运行的容器

启动容器

守护进程方式启动容器

# 命令格式
docker run <参数，可选> [docker_image] [执行的命令]
# 示例
# 让Docker容器在后台以守护形式运行。此时可以通过添加-d参数来实现
docker run -d nginx

启动已终止的容器

在生产过程中，常常会出现运行和不运行的容器，我们使用 start 命令开起一个已关闭的容器

# 命令格式：
docker start [container_id]
# 示例
docker start myubuntu

关闭、删除

关闭容器

在生产中，我们会以为临时情况，要关闭某些容器，我们使用 stop 命令来关闭某个容器

# 命令格式：
docker stop [container_id]
# 示例
docker stop myubuntu

删除容器

正常删除容器

# 命令格式：
docker rm [container_id]
# 示例
# 删除已停止的容器
docker rm myubuntu

强制删除运行容器

# 命令格式：
docker rm -f [container_id]
# 示例
# 删除正在运行的容器
docker rm -f  myubuntu

拓展批量关闭容器

docker rm -f $(docker ps -a -q)

进入、退出容器

创建并进入容器

# 命令格式：
docker run --name [container_name] -it [docker_image] /bin/bash
# 示例
docker run -it --name sswang-nginx nginx /bin/bash

docker 容器启动命令参数详解：

--name:给容器定义一个名称

-i:则让容器的标准输入保持打开。

-t:让docker分配一个伪终端,并绑定到容器的标准输入上

/bin/bash:执行一个命令

退出容器：

方法一：exit

方法二：Ctrl + D

进入后台运行容器

# 命令格式：
docker exec -it 容器id /bin/bash
# 示例
docker exec -it myubuntu /bin/bash

基于容器创建镜像

# 命令格式：
docker commit -m '改动信息' -a "作者信息" [container_id] [new_image:tag]
# 示例
docker commit -m 'mkdir /sswang' -a "sswang" myubuntu myubuntu:last

查看容器日志和信息

查看容器运行日志

# 命令格式：
docker logs [容器id]
# 示例
docker logs myubuntu

查看容器详细信息

# 命令格式：
docker inspect  [容器id]
# 示例
docker inspect  myubuntu
